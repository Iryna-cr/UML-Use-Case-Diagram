# UML Use Case та Sequence Diagram


## UML Use Case Diagram — сценарії взаємодії користувача.

Опис: на діаграмі зображені основні актори та варіанти використання системи:

Користувач може: переглянути фільми, обрати сеанс, забронювати квиток, оплатити квиток, скасувати бронювання.

Адміністратор може: додати фільм, редагувати сеанс


## UML Sequence Diagram — послідовність дій у сценарії «забронювати квиток».

### Опис: 

Ця діаграма відображає послідовність взаємодії між трьома основними учасниками процесу бронювання квитка у кіно:
користувачем, системою бронювання квитків, базою даних.

### Послідовність дій:

1) Користувач надсилає запит до cистеми бронювання квитків із вимогою забронювати місце.
2) Система звертається до ,ази даних, щоб перевірити доступність вибраного місця.
3) Після отримання підтвердження про доступність місця, cистема зберігає дані про нове бронювання у базі даних.
4) Система надсилає підтвердження користувачу, що квиток успішно заброньовано.
   
## Висновок:
Обидві діаграми описують ключову бізнес-функцію системи - бронювання квитка.
Use Case Diagram показує ролі користувачів і сценарії, а Sequence Diagram - технічну послідовність дій між компонентами.
Разом вони забезпечують чітке розуміння логіки роботи системи та взаємодії між її елементами.
